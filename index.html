    PQC Compliance Tiers

    Tier Security Level Compliance Criteria Priority/Action
    Tier-1 Elite Modern best-practice crypto posture TLS 1.2/1.3 only; Strong Ciphers (AES-GCM/ChaCha20); Forward Secrecy (ECDHE); cert >2048-bit; no weak protocols; HSTS enabled Maintain Configuration; periodic monitoring
    Tier-2 Standard Acceptable enterprise configuration TLS 1.2 supported but legacy protocols allowed; Key >2048-bit; Mostly strong ciphers; Forward secrecy optional Improve gradually; disable legacy protocols; standardise cipher suites
    Tier-3 Legacy Weak but still operational TLS 1.0/1.1 enabled; weak ciphers (CBC, 3DES); Forward secrecy missing; Key possibly 1024-bit Remediation required; upgrade TLS stack; rotate certificates
    Critical Insecure / exploitable SSL v2/v3 enabled; Key <1024-bit; weak cipher suites (<112-bit); Known vulnerabilities Immediate action — block or isolate service; patch vulnerabilities
